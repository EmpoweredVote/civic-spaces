# Milestone v3.0: UI/UX Redesign

**Status:** ✅ SHIPPED 2026-04-10
**Phases:** 9–13
**Total Plans:** 11

## Overview

Rebuild the slice UI around Krishna's mockup — geo-grounded hero banners, two-column desktop layout, and a fully-wired community sidebar. Every slice becomes a visually distinct, community-anchored space with live civic data widgets and a zero-warning TypeScript build.

**Design Standard (v3.0 non-negotiable):**
- Beautiful, polished UI is a first-class deliverable — not a nice-to-have
- Light AND dark mode required across all new components
- Desktop and mobile must each be exceptional — separate flows are acceptable if they produce a superior experience for each context
- Reference: `C:\Civic Spaces\Screengrabs\` (6 Krishna mockup screengrabs) for layout, tone, and visual hierarchy
- Use `/ui-ux-pro-max` skill during planning phases to validate component design decisions before implementation

### Phase 9: Hero Banner & Layout Shell

**Goal**: Users see a full-width, identity-rich hero banner above their feed on every slice tab, and desktop users see the two-column layout shell ready to receive a sidebar.

**Depends on**: Phase 8

**Requirements**: HERO-01, HERO-02, HERO-03, LAYOUT-01, LAYOUT-02

**Plans**: 3 plans

Plans:
- [x] 09-01: Two-column layout grid, teal pill tabs, dark mode setup
- [x] 09-02: HeroBanner component + static slice copy data
- [x] 09-03: Hero wiring into AppShell + visual verification

**Details:**

- Tailwind v4 `@custom-variant dark` for class-based dark mode; `--color-civic-teal` theme token
- Two-column CSS grid (65/35) with `min-h-0` chain from `h-screen` to scrollRef for scroll preservation
- CSS-hidden feed panels (not unmounted) preserves React Query cache, Realtime subscriptions, and DOM scroll across tab switches
- Teal pill active tab: `bg-teal-600 rounded-lg px-4 py-2` — matches Krishna mockup rectangular capsule, not border-b-2
- `HeroBanner.tsx` pure presentational (no hooks, no state) — all data via props
- `SLICE_COPY` static data map with tagline/description/defaultPhoto for all 6 slice types
- HeroBanner sits above CSS-hidden tab divs — always rendered, swaps content via `activeTab` prop, never remounts
- Feed panel wrapper uses `flex-1 overflow-hidden min-h-0` — hero takes natural height, feed scrolls independently

### Phase 10: Photos & Storage

**Goal**: Jurisdiction-specific hero photos are stored in Supabase Storage and served via CDN URL into the Phase 9 hero banner, giving Bloomington pilot slices authentic local imagery.

**Depends on**: Phase 9

**Requirements**: PHOTO-01, PHOTO-02

**Plans**: 2 plans

Plans:
- [x] 10-01: Storage bucket + migration + photo_url column + HeroBanner wiring
- [x] 10-02: Wikipedia hero images + jurisdiction name resolution + tab label renames + Unified backfill

**Details:**

- `slice-photos` Supabase Storage bucket with `public=true` + explicit SELECT RLS policy (bucket flag alone causes 403)
- `photo_url` nullable text column on `civic_spaces.slices` as curated DB override layer
- **Approach pivot:** Wikipedia REST API replaces Supabase Storage uploads — `/api/rest_v1/page/summary/{title}` returns geo-appropriate images (state capitols, county courthouses) for free, no auth required
- `geoidToWiki.ts` — FIPS decoder: (sliceType, geoid) → Wikipedia article title. Full 50-state table + Indiana county fast path. federal/state → state capitol, local → county courthouse
- `useWikiHeroImage.ts` — session-cached Map (keyed by sliceType|geoid) prevents re-fetch on tab switch
- `useJurisdictionName.ts` — Census Bureau API (`api.census.gov`) for human-readable county/place names; no API key required
- `ActiveHeroBanner` extracted wrapper — calls hooks at component level, passes to pure HeroBanner. Photo resolution: `photoUrl (DB) > wikiPhotoUrl > copy.defaultPhoto > null`
- Tab labels: neighborhood→"Local", local→"County" (reflects civic jurisdiction levels better than TIGER/Line technical terms)
- Unified slice backfilled to all existing connected users (BloomingtonVoter and Kades) via production migration

## Milestone Summary

**Key Decisions:**

- Wikipedia REST API over Supabase Storage uploads — zero-upload, zero-auth, returns geo-appropriate photos for any US state or county automatically
- Hook hoisting at AppShell level — all 6 feed panels mount simultaneously (CSS-hidden pattern); hooks inside panels would fire 6x each
- Anti-partisan policy enforced at type layer (`party` field omitted from `PoliticianFlatRecord`), render layer (never rendered in widgets), and color layer (purple #7c3aed, no red/blue)
- Sidebar width 18% (not 35%) — feed content should dominate; sidebar is supplemental
- Volunteer tab sidebar hidden entirely — reserved for future task assignment modal, civic widgets not relevant
- React.lazy named-export shim pattern: `lazy(() => import('./X').then((m) => ({ default: m.X })))` — required because React.lazy expects a default export
- `min-h-0` must be on every flex/grid ancestor from `h-screen` down to `scrollRef` target or grid expands to intrinsic content height and scroll stops working

**Issues Resolved:**

- Phase 10 approach pivot: Upload script + storage seeds replaced by Wikipedia REST API (cleaner, no service-role keys, better photos)
- PostgREST schema cache reload needed after `photo_url` DDL (NOTIFY pgrst, 'reload schema' + ~2 min wait)
- Missing Phase 7 migration in production (unified/volunteer CHECK constraint) applied manually before Unified backfill
- Federal slice Wikipedia: congressional district pages are stubs with no images → falls back to state capitol article
- Hero banner text overflow on mobile: description paragraph hidden on mobile to prevent overflow
- Sidebar API URL: `accounts.empowered.vote/api` → `api.empowered.vote/api` (confirmed from CompassV2 source)
- Treasury Tracker excluded from LIVE_TOOLS (connection refused at build time)

**Issues Deferred:**

- CORS confirmation for `civicspaces.empowered.vote` in `api.empowered.vote` CORS allowlist (hooks written, runtime validation pending)
- Animated hero banner transitions on tab switch (VIS-01 → v4.0+)
- Skeleton loading states for sidebar widgets (VIS-02 → v4.0+)

**Technical Debt Incurred:**

- Phase 10/11 used UAT.md format instead of gsd-verifier VERIFICATION.md (process only, no functional gap)
