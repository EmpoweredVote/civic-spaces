## Tech Debt

### Phase 08 — Profile Pages (2 items, non-blocking)

- **Dead prop:** `MutualFriendsList` accepts `friendCount` as a prop but the value is never rendered — the heading uses `mutualFriends.length` (mutual count, which is correct). The prop is harmless but creates dead interface surface; can be removed in a future cleanup.
- **Vestigial prop:** `onNavigateToThread` is optional in `NotificationListProps` and always passed at both call sites in `NotificationBell`, but is aliased as `_onNavigateToThread` inside the component and never invoked. All reply navigation routes through `onNavigateToSliceThread`. The legacy prop is dead interface surface; can be removed in a future cleanup.

Neither item affects runtime behavior or any requirement.
