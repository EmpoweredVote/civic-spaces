**Create src/hooks/useFlag.ts:**

- `useMyFlags(userId: string | null)` — useQuery that fetches all flags where reporter_id = userId from the `flags` view. Returns a Set of post_ids for O(1) lookup. queryKey: `['my-flags', userId]`. Enabled only when userId is truthy.

- `useFlagPost()` — useMutation that inserts into `flags` view with {post_id, reporter_id, category, detail}. On success: invalidate `['my-flags']` query, show `toast.success('Thanks, we will review this')` via sonner. On error: if error code is '23505' (duplicate), treat as success — show same toast, no error. For other errors, show `toast.error('Failed to submit flag')`.

**Create src/hooks/useBlock.ts:**

- `useBlockedUsers(userId: string | null)` — useQuery fetching all rows from `blocks` where blocker_id = userId. Returns array of Block objects AND a Set of blocked_ids for O(1) lookup. queryKey: `['blocked-users', userId]`. Enabled only when userId is truthy.

- `useBlockUser()` — useMutation inserting into `blocks` view with {blocker_id, blocked_id}. On success: invalidate `['blocked-users']`, invalidate `['feed']` and `['boosted-feed']` queries (to refetch filtered feed), show `toast.success('User blocked')`.

- `useUnblockUser()` — useMutation deleting from `blocks` where blocker_id and blocked_id match. On success: same invalidations, show `toast.success('User unblocked')`.

**Create src/hooks/useModQueue.ts:**

- `useModQueue(filters?: { status?: string, category?: string })` — useQuery calling `supabase.rpc('get_mod_queue', { p_status: filters?.status ?? 'pending', p_category: filters?.category ?? null })`. queryKey: `['mod-queue', filters]`. staleTime: 0, refetchInterval: 30000 (polling, NOT Realtime — research says Realtime respects RLS and won't work for mod queue). Returns ModQueueItem[].

- `useIsModerator(userId: string | null)` — useQuery fetching from `moderators` view where user_id = userId. Returns boolean. queryKey: `['is-moderator', userId]`. Enabled only when userId is truthy.

- `useModAction()` — useMutation calling `supabase.rpc('mod_action', { p_action, p_post_id, p_user_id, p_notes })`. On success: invalidate `['mod-queue']`, show `toast.success` with action-specific message ('Content removed', 'Flags dismissed', 'Warning sent', 'User suspended').

**Modify src/hooks/useFeed.ts:**

Replace the direct table query with the `get_feed_filtered` RPC. Change the `fetchFeedPage` function to call:
```typescript
const { data, error } = await supabase.rpc('get_feed_filtered', {
  p_slice_id: sliceId,
  p_limit: PAGE_SIZE,
  p_cursor_at: cursor?.created_at ?? null,
  p_cursor_id: cursor?.id ?? null,
})
```
The RPC returns the same columns as the current query but with block filtering built in. After getting the raw posts, the existing author-enrichment logic (fetching connected_profiles) stays the same — just change the data source from table query to RPC call.

**Modify src/hooks/useBoostedFeed.ts:**

Same pattern — replace the existing `get_boosted_feed` RPC call with `get_boosted_feed_filtered`. The return signature is identical, just the function name changes. The block filtering is handled server-side.
