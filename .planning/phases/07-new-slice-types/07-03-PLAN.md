  <name>Task 1: Extend useAllSlices to return all slice types with sibling_index</name>
  <files>src/hooks/useAllSlices.ts</files>
  <action>
Modify `useAllSlices.ts` to return ALL slice types (not just geo) and include `siblingIndex` in `SliceInfo`:

1. **Remove the GEO_SLICE_TYPES constant entirely** (line 5: `const GEO_SLICE_TYPES: SliceType[] = [...]`). It is no longer needed.

2. **Add `sibling_index` to the select query** (line 36):
   Change:
   ```typescript
   .select('id, slice_type, geoid, current_member_count')
   ```
   To:
   ```typescript
   .select('id, slice_type, geoid, current_member_count, sibling_index')
   ```

3. **Remove the GEO_SLICE_TYPES filter in the loop** (line 45). Change the loop body from:
   ```typescript
   if (GEO_SLICE_TYPES.includes(sliceType)) {
     slices[sliceType] = { ... }
   }
   ```
   To (no filter — include all types):
   ```typescript
   slices[sliceType] = {
     id: row.id,
     sliceType,
     geoid: row.geoid,
     memberCount: row.current_member_count,
     siblingIndex: row.sibling_index,
   }
   ```

4. **Fix `hasJurisdiction` semantics.** Currently `hasJurisdiction` is `true` if ANY memberships exist. After this change, a user with only a Unified slice (no geo) would get `hasJurisdiction: true`, which would hide the `NoJurisdictionBanner` incorrectly (they still need geo assignment). Change the logic:

   Replace:
   ```typescript
   const hasJurisdiction = memberships != null && memberships.length > 0
   ```

   Keep that line for the early return (if no memberships at all, return empty). But after building the slices map, derive `hasJurisdiction` from whether any GEO slice exists:
   ```typescript
   const hasGeoSlices = !!slices['federal']  // Federal is always assigned when jurisdiction exists
   return { slices, hasJurisdiction: hasGeoSlices }
   ```

   This way `hasJurisdiction` means "user has geo jurisdiction" (controls NoJurisdictionBanner), while `slices` still contains unified/volunteer entries for users without geo.

   IMPORTANT: The early return for `memberships.length === 0` should still return `{ slices: {}, hasJurisdiction: false }`. The `hasGeoSlices` check only applies when memberships exist.

Do NOT rename `hasJurisdiction` to `hasAnySlice` — the field name is used in AppShell and the semantics (geo jurisdiction check) are correct after this change.
