<objective>
Build the slice assignment Express service that runs on Render. On each login, the frontend calls this service with the user's Bearer token. The service verifies the token, fetches jurisdiction GEOIDs from the accounts API, and upserts the user into all four slice_members entries (federal, state, local, neighborhood). It also syncs the user's display_name into connected_profiles for fast JOIN reads.

Purpose: This is the glue between the accounts system and the civic_spaces schema. Without it, users have no slice memberships and cannot see any content.
Output: A complete Express service ready to deploy on Render, with token verification, accounts API integration, and idempotent slice assignment with overflow handling.
</objective>

    1. Create `src/services/sliceAssigner.ts`:

       **Supabase client setup:**
       - Create a service-role Supabase client (bypasses RLS for privileged writes):
         ```typescript
         import { createClient } from '@supabase/supabase-js'
         ```

       **Export `assignUserToSlices(userId: string, jurisdiction: AccountData['jurisdiction'])`:**
       - Define the four assignment mappings:
         ```typescript
         const assignments: { sliceType: string; geoid: string }[] = [
           { sliceType: 'federal',      geoid: jurisdiction.congressional },
           { sliceType: 'state',        geoid: jurisdiction.state_senate },
           { sliceType: 'local',        geoid: jurisdiction.county },
           { sliceType: 'neighborhood', geoid: jurisdiction.school_district },
         ]
         ```
         Note: Use `state_senate` for state slice (not state_house). This was decided in RESEARCH.md.
       - For each assignment, call `findActiveSliceForGeoid(sliceType, geoid)` then `upsertSliceMember(userId, sliceId)`
       - Collect results and return a summary: `{ assigned: string[] }` listing the slice types successfully assigned

       **`findActiveSliceForGeoid(sliceType: string, geoid: string): Promise<string>`:**
       - Query: `SELECT id FROM civic_spaces.slices WHERE slice_type = $1 AND geoid = $2 AND current_member_count < 6000 ORDER BY sibling_index ASC LIMIT 1`
       - Use the Supabase client with `.schema('civic_spaces')`:
         ```typescript
         const { data, error } = await supabase
           .schema('civic_spaces')
           .from('slices')
           .select('id')
           .eq('slice_type', sliceType)
           .eq('geoid', geoid)
           .lt('current_member_count', 6000)
           .order('sibling_index', { ascending: true })
           .limit(1)
           .single()
         ```
       - If no slice exists (first user for this GEOID): create the initial slice with `sibling_index = 1` and return its ID
       - If all existing slices are full: call `findOrCreateSiblingSlice(sliceType, geoid)`

       **`findOrCreateSiblingSlice(sliceType: string, geoid: string): Promise<string>`:**
       - Find the max `sibling_index` for this (geoid, slice_type):
         ```typescript
         const { data } = await supabase
           .schema('civic_spaces')
           .from('slices')
           .select('sibling_index')
           .eq('slice_type', sliceType)
           .eq('geoid', geoid)
           .order('sibling_index', { ascending: false })
           .limit(1)
           .single()
         ```
       - Insert a new slice with `sibling_index = maxIndex + 1`
       - Use the UNIQUE constraint on (geoid, slice_type, sibling_index) to handle race conditions: if two concurrent requests try to create the same sibling, the second INSERT will conflict. Catch the conflict error and re-query for the existing sibling.
       - Return the new (or existing) slice ID

       **`upsertSliceMember(userId: string, sliceId: string): Promise<void>`:**
       - Upsert into slice_members:
         ```typescript
         const { error } = await supabase
           .schema('civic_spaces')
           .from('slice_members')
           .upsert(
             { user_id: userId, slice_id: sliceId },
             { onConflict: 'user_id,slice_id', ignoreDuplicates: true }
           )
         ```
       - If error message includes 'slice_full' (from the BEFORE INSERT trigger):
         a. Call `findOrCreateSiblingSlice` for this slice's type and geoid
         b. Retry `upsertSliceMember` with the sibling slice ID
         c. Limit retries to 3 to prevent infinite loops
       - On other errors: throw

       **Export `upsertConnectedProfile(userId: string, displayName: string, accountStanding: string): Promise<void>`:**
       - Upsert into connected_profiles:
         ```typescript
         const { error } = await supabase
           .schema('civic_spaces')
           .from('connected_profiles')
           .upsert(
             { user_id: userId, display_name: displayName, account_standing: accountStanding },
             { onConflict: 'user_id' }
           )
         ```
       - This syncs the display_name from accounts on every login so the local cache stays fresh
