- `useRemoveFriend()` — mutation that deletes the friendship row. Invalidates ['relationship'] and ['friends'] queries.

Import friendshipKey and friendshipStatus from src/lib/friendship.ts. Import useAuth for current userId.

**Create src/hooks/useFriends.ts:**
- `useFriendsList()` — queries the friendships table for all rows where current user is user_low OR user_high. Returns two arrays: `friends` (status === 'FRIEND') and `pendingReceived` (requests where current user is the recipient — i.e., status is REQ_LOW and user is user_high, or status is REQ_HIGH and user is user_low). For each friendship row, extract the other user's ID and fetch their profile (display_name, avatar_url, tier) from connected_profiles in a batch query. QueryKey: ['friends']. StaleTime: 2 minutes.

**Create src/hooks/useMemberDirectory.ts:**
- `useMemberDirectory(sliceId)` — paginated query of slice members. First fetches user_ids from slice_members for the given sliceId, then fetches connected_profiles for those IDs. Returns array of profiles. Uses useInfiniteQuery with cursor-based pagination (offset-style is acceptable here since directory is alphabetically sorted and stable). Limit 50 per page.

**Create src/hooks/useMemberSearch.ts:**
- `useMemberSearch(term, crossSlice, sliceId)` — as shown in 03-RESEARCH.md Pattern 7. When crossSlice is false, uses two-query pattern (fetch slice_members IDs first, then filter profiles). When crossSlice is true, queries all connected_profiles with ilike filter. Always applies .limit(50). Enabled only when term.length >= 2. StaleTime 30 seconds.

**Create src/components/FriendsList.tsx:**
Full-screen panel (or tab content) showing the user's friends and pending requests.

Layout:
1. If pendingReceived is non-empty: "Pending Requests" section header, then list of profile rows each with Accept/Decline buttons. Accept calls useAcceptFriendRequest; Decline calls useRemoveFriend (deletes the row).
2. "Friends" section header, then list of friend profile rows. Each row shows avatar, display name, EmpoweredBadge if applicable. Tapping a row opens UserProfileCard for that user.
3. Empty state when no friends: centered text like "No friends yet. Find people in the Member Directory!"
4. Use react-loading-skeleton for loading state.

**Create src/components/MemberDirectory.tsx:**
Panel with a search input at top, a "Search beyond your slice" toggle, and a scrollable member list.

Layout:

3. When search term is empty or < 2 chars: show directory list from useMemberDirectory.
4. When search term >= 2 chars: show search results from useMemberSearch.
5. Each member row: avatar, display name, EmpoweredBadge if empowered. Tapping opens UserProfileCard.
6. Loading: react-loading-skeleton rows.
