  <name>Task 1: useNotifications hook with Realtime and mark-read mutations</name>
  <files>src/hooks/useNotifications.ts</files>
  <action>
Create `src/hooks/useNotifications.ts` following the established hook patterns (useBoostedFeed, useFriendship):

**useNotifications():**
- Import supabase client, useAuth (for userId), useQueryClient, useQuery, useEffect
- Query: `supabase.from('notifications').select('*').order('updated_at', { ascending: false }).limit(50)`
- queryKey: `['notifications', userId]`
- enabled: `!!userId`
- staleTime: 30_000 (30 seconds)
- Realtime: own channel `notifications-${userId}`, subscribe to `postgres_changes` on `civic_spaces.notifications` table filtered by `recipient_id=eq.${userId}`, event `*` — invalidate `['notifications', userId]` on any change
- Cleanup: remove channel on unmount
- Derive `unreadCount`: `Math.min((data ?? []).filter(n => !n.is_read).length, 99)`
- Return: `{ notifications: data ?? [], unreadCount, isLoading }`

**useMarkNotificationRead():**
- useMutation: `supabase.from('notifications').update({ is_read: true }).eq('id', notificationId)`
- onSuccess: invalidate `['notifications', userId]`

**useMarkAllNotificationsRead():**
- useMutation: `supabase.from('notifications').update({ is_read: true }).eq('is_read', false)`
- RLS enforces recipient_id scoping, but add `.eq('recipient_id', userId!)` for defense-in-depth
- onSuccess: invalidate `['notifications', userId]`

All three hooks exported as named exports. Type the query response as `Notification[]` from `src/types/database.ts`.
  </action>
  <verify>`grep -c "useNotifications\|useMarkNotificationRead\|useMarkAllNotificationsRead" src/hooks/useNotifications.ts` returns 3+ (all three hooks defined)</verify>
  <done>Three hooks exported: useNotifications (query + realtime), useMarkNotificationRead, useMarkAllNotificationsRead</done>
