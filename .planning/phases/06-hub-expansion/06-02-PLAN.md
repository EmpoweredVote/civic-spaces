  <action>
  The CSS hidden pattern from 06-01 already keeps all feeds mounted, which means the browser preserves DOM scroll position natively for the inner scroll container. However, some browsers reset scroll on visibility toggle, and the `hidden` class uses `display: none` which removes elements from layout.

  **Approach: Use `visibility: hidden` + position tricks OR save/restore scroll explicitly.**

  The reliable approach is explicit save/restore:

  1. **SliceFeedPanel.tsx** — Expose scroll container ref:
     - Add an optional `scrollRef` prop: `scrollRef?: React.RefObject<HTMLDivElement | null>`
     - Attach it to the feed's scrollable div (the one with `overflow-y-auto`, currently: `<div className={activePostId ? 'hidden' : 'flex flex-col h-full overflow-y-auto'}>`)
     - This ref lets AppShell read/write scrollTop on that element

  2. **AppShell.tsx** — Save and restore scroll positions:
     - Add a ref: `const scrollPositions = useRef<Record<string, number>>({})`
     - Create refs for each tab's scroll container: `const scrollRefs = useRef<Record<string, React.RefObject<HTMLDivElement | null>>>({})`
     - For each geo tab's SliceFeedPanel, create a stable ref and pass it as `scrollRef`
     - On tab change (in the `setActiveTab` handler or via a useEffect watching `activeTab`):
       a. BEFORE switching: save current scroll position: `scrollPositions.current[prevTab] = scrollRefs.current[prevTab]?.current?.scrollTop ?? 0`
       b. AFTER switching: restore new tab's position: `scrollRefs.current[newTab]?.current?.scrollTo(0, scrollPositions.current[newTab] ?? 0)`
     - Use a `useEffect` that fires on `activeTab` change to handle the restore. Use a `useRef` to track the previous tab for the save.
     - Timing: The restore must happen after the CSS `hidden` class is removed so the element has layout. Use `requestAnimationFrame` for the restore to ensure the browser has rendered.

  **Critical detail from research:** The scroll ref must target the INNER scrollable div inside SliceFeedPanel (the `overflow-y-auto` div), NOT the outer AppShell wrapper. The outer wrapper may also have overflow, causing confusion about which element to target.

  **Critical detail:** The `hidden` Tailwind class sets `display: none`, which resets scrollTop to 0 on some browsers. The save must happen BEFORE the class changes. Sequence:
  1. Save old tab scrollTop
  2. Update activeTab state (triggers re-render, toggling hidden classes)
  3. After paint, restore new tab scrollTop via requestAnimationFrame

  Implement this as a wrapper around setActiveTab:
  ```typescript
  const handleTabChange = useCallback((newTab: TabKey) => {
    // Save current scroll position before switching
    const currentRef = scrollRefs.current[activeTab]
    if (currentRef?.current) {
      scrollPositions.current[activeTab] = currentRef.current.scrollTop
    }
    setActiveTab(newTab)
  }, [activeTab])
  ```
  Then a useEffect on activeTab change:
  ```typescript
  useEffect(() => {
    requestAnimationFrame(() => {
      const ref = scrollRefs.current[activeTab]
      if (ref?.current) {
        ref.current.scrollTop = scrollPositions.current[activeTab] ?? 0
      }
    })
  }, [activeTab])
  ```
