  <name>Task 1: Create useNotificationRouting hook</name>
  <files>src/hooks/useNotificationRouting.ts</files>
  <action>
  Create `src/hooks/useNotificationRouting.ts` that resolves a notification's reference_id (which is a post_id for reply notifications) to a TabKey.

  The hook:
  - Accepts the allSlices map (from useAllSlices) as a parameter: `slices: Partial<Record<SliceType, SliceInfo>>`
  - Exports a function `resolveTabForPost(postId: string): Promise<TabKey>`:
    1. Query `supabase.schema('civic_spaces').from('posts').select('slice_id').eq('id', postId).single()`
    2. Get the `slice_id` from the result
    3. Reverse-lookup: iterate over the `slices` map entries and find which entry has `id === slice_id`
    4. Return the matching SliceType as TabKey
    5. If no match found (post is in a slice the user isn't a member of, or query fails), return `'federal'` as fallback
  - This is NOT a React Query hook — it's a simple async utility function wrapped in a hook that closes over the slices map. No caching needed since it's called on tap (one-off).

  Export pattern:
  ```typescript
  export function useNotificationRouting(slices: Partial<Record<SliceType, SliceInfo>>) {
    const resolveTabForPost = useCallback(async (postId: string): Promise<TabKey> => {
      // ... implementation
    }, [slices])
    return { resolveTabForPost }
  }
  ```
  </action>
  <verify>Run `npx tsc --noEmit` — no type errors. File exists and exports useNotificationRouting.</verify>
  <done>useNotificationRouting hook resolves a post_id to the correct TabKey by looking up the post's slice_id and matching it against the user's slices map.</done>
</task>
