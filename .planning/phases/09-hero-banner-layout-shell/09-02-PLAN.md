Create `src/components/HeroBanner.tsx` as a presentational component. It receives slice data as props and renders the hero banner matching the Krishna mockup layout.

**Props interface:**
```typescript
interface HeroBannerProps {
  sliceType: SliceType
  sliceName: string
  geoid: string
  memberCount: number
  siblingIndex: number
}
```

**Visual structure (from mockups):**
1. Full-width container with fixed aspect ratio: `aspect-[16/5]` on desktop (`md:`), `aspect-[16/9]` on mobile
2. Background image from `SLICE_COPY[sliceType].placeholderPhoto` via inline `style={{ backgroundImage }}` — guard against null/undefined (show solid gray fallback `bg-gray-700 dark:bg-gray-800`)
3. Dark gradient overlay from bottom: `bg-gradient-to-t from-black/70 via-black/40 to-transparent` — this ensures white text is readable over any photo
4. Text content positioned at bottom-left with padding `p-6 md:p-8`

**Text hierarchy (bottom-left, white on gradient):**
- Slice name: `text-2xl md:text-3xl font-bold text-white`
- Tagline: `text-sm md:text-base text-white/90 mt-1`
- Pill badges row: `flex flex-wrap gap-2 mt-3`
  - Each pill: `px-3 py-1 text-xs font-medium rounded-full bg-white/20 text-white backdrop-blur-sm`
  - Pills to show:
    1. Level pill: map sliceType to human label — "Neighborhood Level", "Local Level", "State Level", "Federal Level", "Unified", "Volunteer"
    2. Jurisdiction pill: show `geoid` (in Phase 10 this will be resolved to a name; for now show raw geoid)
    3. Member count pill: `${memberCount.toLocaleString()} verified residents`
    4. Slice number pill: `Slice ${siblingIndex}` (do NOT show "of Y" — siblingTotal is not available in SliceInfo; add a `// TODO Phase 10: add siblingTotal when available` comment)
- Description: `text-xs md:text-sm text-white/80 mt-3 max-w-2xl leading-relaxed`

**Important implementation details:**
- Import `SLICE_COPY` from `../lib/sliceCopy`
- Import `SliceType` from `../types/database`
- Use `bg-cover bg-center` for the background image
- The component is `relative` with `overflow-hidden` and the gradient overlay is an `absolute inset-0` div
- Text content is in a `relative z-10` div to sit above the gradient
- Add `rounded-xl` and `mx-4 mt-4 md:mx-0 md:mt-0` for slight card-like inset on mobile, flush on desktop
- The component should NOT manage any state — pure presentational

**Dark mode:** The hero banner is already dark (photo with dark gradient), so it looks good in both modes. Add `dark:ring-1 dark:ring-white/10` to the outer container for a subtle border in dark mode.
